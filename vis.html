3D Model & Circuit

The selected concept was developed into a full 3D assembly and circuit schematic.

3D Model Parts & Assembly

Individual components — chassis, linkage arms, cam shafts, leg sets, and support brackets — were modelled in CAD and assembled virtually before physical fabrication began. This step caught fitment issues early and confirmed part dimensions matched the Bill of Materials.

Circuit Diagram

The electronics schematic maps the L298N motor driver to the left and right BO motors, the IR obstacle sensor input, and the microcontroller's logic. Power rails are separated: motor power at +12V and logic power at +5V. Enable pins (ENA, ENB) support optional PWM speed control.

L298N Motor Driver Connections

IN1 and IN2 control the left motor; IN3 and IN4 control the right motor. ENA and ENB are the PWM enable pins. Motor power connects to +12V; logic power to +5V from the microcontroller.

Obstacle Avoidance Flowchart

The robot's control logic — from power-on to continuous obstacle detection and avoidance.

1

System Initialization

Power ON the microcontroller, initialize the L298N motor driver, set BO motors (Left & Right) as OUTPUT, set IR sensor as INPUT, and stop both motors.

2

Main Loop

The program enters a continuous loop that repeats until the system is powered off. All sensing and actuation decisions happen within this loop.

3

Read IR Obstacle Sensor

Read the IR sensor output and check whether an obstacle is present in the robot's path.

4A

Path Clear

If no obstacle: L298N drives both motors FORWARD. The robot moves straight and continues checking the IR sensor.

4B

Obstacle Ahead

If obstacle detected: stop both motors with a small delay before executing the avoidance action.

5

Avoidance Action

Rotate the robot to avoid the obstacle. Turn Right: Left motor OFF, Right motor ON. Turn Left: Right motor OFF, Left motor ON.

6

Check Path Again

Read the IR sensor again. If the path is still blocked, continue rotating. Once clear, resume normal forward movement.

Components Used

The core hardware that powers the obstacle-avoidance system.

⚙️

2× BO Motors

Left & right drive motors providing independent wheel control for forward motion and turning.

📡

IR Obstacle Sensor

Detects objects in front of the robot and sends a HIGH/LOW signal to the microcontroller.

🔌

L298N Motor Driver

Bridges the microcontroller's logic signals to the higher-current motors. Supports PWM speed control via ENA/ENB pins.

🧠

Microcontroller

The brain of the system — runs the main loop, reads sensor inputs, and sends commands to the motor driver.
