Claude Token Efficiency Skill Tree Level 1: Context Filtering Goal: Send only what Claude needs. Train yourself to ask: What is the actual problem? Which files are involved? Which lines matter? What can be omitted? Exercise Take a 3,000-line file and try explaining the issue in under 100 words. Level 2: Constraint Writing Goal: Prevent unwanted changes. Learn to specify: DO NOT CHANGE: - Theme - Colors - Fonts - Existing functionality CHANGE ONLY: - Navbar spacing Good constraints reduce retries. Exercise Rewrite vague requests into precise constraints. Example: Bad: Make it better. Good: Improve mobile responsiveness without changing desktop layout. Level 3: Prompt Templating Goal: Stop rewriting instructions. Build reusable templates. Examples: Bug Fix Template ROLE: Senior Frontend Engineer TASK: Fix the issue described below. ISSUE: [Issue] DO NOT CHANGE: [List] OUTPUT: Only modified code. Code Review Template Analyze this code for: - Bugs - Performance issues - Accessibility issues Return findings only. Level 4: Scope Isolation Goal: Break large problems into small ones. Instead of: Build my entire portfolio. Use: Analyze structure. Fix navigation. Improve Section 06. Optimize animations. Generate final code. Exercise Split a large task into 5–10 independent tasks. Level 5: Output Control Goal: Reduce Claude's response size. Use phrases like: Return only HTML. No explanations. No summaries. No repeated code. Exercise Compare outputs with and without these instructions. Level 6: Context Compression Goal: Convert huge contexts into small summaries. Prompt: Compress this project into a reusable specification under 500 tokens. Preserve constraints and architecture. Reuse the compressed version later. Exercise Turn a long report into a one-page specification. Level 7: Incremental Editing Goal: Modify instead of regenerate. Avoid: Rewrite the whole file. Prefer: Modify only the uploaded section. Return a patch. Exercise Practice requesting changes to only 10–20 lines. Level 8: Conversation Memory Management Goal: Use the same chat strategically. In one conversation: Upload assets once. Define constraints once. Reuse previous outputs. Avoid reopening new chats for the same project unless necessary. Exercise Complete an entire project while never re-uploading the same instructions. Level 9: Token Budgeting Goal: Think before sending. Create a mental checklist: Input Budget Essential files only Essential screenshots only Constraints only Output Budget Exact format needed No explanations No duplicate content Exercise Estimate whether a prompt is: Low (<500 tokens) Medium (500–2,000) High (>2,000) Level 10: AI Workflow Design Goal: Design an efficient AI-assisted workflow. For example: Portfolio Workflow Phase 1 Analyze files. Generate project specification. Phase 2 Fix individual sections. Phase 3 Review for consistency. Phase 4 Generate final deliverables. Claude becomes part of your process rather than a one-off generator. Advanced Skills Diff Thinking Request: Show only additions and modifications. Do not repeat unchanged code. Failure Diagnosis If Claude fails: Ask: What information was missing? What constraints conflicted? What caused the incomplete output? Fix the prompt instead of retrying blindly. Prompt Refactoring Treat prompts like code. Version them: portfolio_prompt_v1 portfolio_prompt_v2 portfolio_prompt_v3 Improve them over time. Mastery Levels Beginner Uses templates Avoids unnecessary explanations Intermediate Compresses context Isolates scope Controls output Advanced Designs multi-step workflows Uses diffs and patches Diagnoses failures efficiently Expert Can complete complex projects with minimal retries and predictable token usage. Thinks in terms of context architecture, not just prompts.
